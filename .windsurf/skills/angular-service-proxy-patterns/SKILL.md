# Angular Service Proxy Patterns

## CRITICAL Rule

**NEVER manually create `*ServiceProxy` classes or DTO classes** in the Angular project.

All service proxies are **auto-generated by NSwag** from the backend Swagger/OpenAPI specification.

## Generating Proxies

1. Run the `Web.Host` project (backend must be running)
2. Execute in the `angular/` directory:
   ```bash
   npm run nswag
   ```
   Or on Windows: `angular/nswag/refresh.bat`

3. Generated output: `angular/src/shared/service-proxies/service-proxies.ts`

## MANDATORY: Registering New Proxies as Providers

After NSwag regenerates proxies, **every new `*ServiceProxy` class must be manually added** to `angular/src/shared/service-proxies/service-proxy.providers.ts`.

Without this step, Angular throws `NG0201: No provider found for _XxxServiceProxy` at runtime.

```typescript
// service-proxy.providers.ts
import * as ApiServiceProxies from './service-proxies';

export const SERVICE_PROXY_PROVIDERS = [
    // ... existing entries ...
    ApiServiceProxies.ProductServiceProxy,
    ApiServiceProxies.CustomerServiceProxy,
    ApiServiceProxies.SaleServiceProxy,
    // Add one line per new service proxy here
];
```

import {

```

export class ProductsComponent extends AppComponentBase {

```

## Common Usage Patterns

### Calling a List Method

NSwag generates `[HttpPost]` list methods with a **single DTO input parameter**. Always use the DTO constructor — never pass individual arguments:

```typescript
this._productService
    .getProducts(
        new GetProductsInput({
            filter: this.filterText(),
            sorting: this.primengTableHelper.getSorting(this.dataTable()),
            maxResultCount: this.primengTableHelper.getMaxResultCount(this.paginator(), event),
            skipCount: this.primengTableHelper.getSkipCount(this.paginator(), event),
        })
    )
    .pipe(finalize(() => this.primengTableHelper.hideLoadingIndicator()))
    .subscribe((result) => {
        this.primengTableHelper.records = result.items;
        this.primengTableHelper.totalRecordsCount = result.totalCount;
    });
```

### Calling a Create/Update Method

```typescript
this._productService
    .createProduct(input)
    .pipe(finalize(() => this.saving.set(false)))
    .subscribe(() => {
        this.notify.info(this.l('SavedSuccessfully'));
        this.close();
        this.modalSave.emit();
    });
```

### Calling a Delete Method

```typescript
this._productService.deleteProduct(product.id).subscribe(() => {
    this.getProducts();
    abp.notify.success(this.l('SuccessfullyDeleted'));
});
```

### Input DTO Construction

```typescript
// Use constructor with object literal
const input = new GetProductsInput({
    filter: this.filter(),
    maxResultCount: 10,
    skipCount: 0,
});
```

## finalize Pattern

Always use `finalize` to clean up loading states, regardless of success or error:

```typescript
this._service.method(input)
    .pipe(finalize(() => {
        this.primengTableHelper.hideLoadingIndicator();
        // or: this.saving.set(false);
    }))
    .subscribe((result) => { ... });
```

## Checklist

- [ ] New `*ServiceProxy` added to `SERVICE_PROXY_PROVIDERS` in `service-proxy.providers.ts`
- [ ] Service proxy imported from `@shared/service-proxies/service-proxies`
- [ ] No manually created proxy classes
- [ ] `inject(ServiceProxy)` for DI
- [ ] List methods called with a single `new XxxInput({...})` DTO object
- [ ] `getMaxResultCount(paginator, event)` and `getSkipCount(paginator, event)` receive the `event` arg
- [ ] `finalize()` for loading state cleanup
- [ ] `pipe()` used for RxJS operators
