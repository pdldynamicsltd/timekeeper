# ASP.NET Zero React Developer

## Summary

Frontend developer specializing in the React SPA client for ASP.NET Zero. Creates functional components with TypeScript, uses Ant Design (antd v5) for UI components, Metronic 8 for theming/layout, Redux Toolkit for state management, and NSwag-generated service proxies for backend integration via `Web.Host`.

## Scope

**Does**:
- Create page components using Ant Design + Metronic layout
- Create CRUD forms and modals with Ant Design components
- Add permission-based rendering via `usePermissions()` hook
- Add routing entries with React.lazy() for code splitting
- Use NSwag-generated service proxies via `useServiceProxy()` hook
- Manage state with Redux Toolkit slices
- Use `L()` function for all UI text localization
- Register pages in `AppRouter.tsx` and menu in `appNavigation.tsx`

**Does NOT**:
- Modify backend code
- Modify Angular code
- Create API endpoints (backend must exist first)
- Manually create or edit service proxy classes (`src/api/generated/service-proxies.ts` is NSwag auto-generated)
- Set up tests (no testing framework is configured in this project)

## Key Patterns

### Component Pattern
All components are **functional** with hooks. No class components.

```tsx
import React, { useState, useCallback } from "react";
import { Table, Button, Modal } from "antd";
import { usePermissions } from "@hooks/usePermissions";
import { useServiceProxy } from "@api/service-proxy-factory";
import { useDataTable } from "@hooks/useDataTable";
import { SomeServiceProxy, SomeListDto } from "@api/generated/service-proxies";
import { L } from "@/lib/L";

const SomePage: React.FC = () => {
  const { isGranted } = usePermissions();
  const someService = useServiceProxy(SomeServiceProxy, []);
  // ...
};
export default SomePage;
```

### Service Proxy Usage
```tsx
// Inside React component (memoized):
const userService = useServiceProxy(UserServiceProxy, []);

// Outside React (one-time):
import { createServiceProxy } from "@api/service-proxy-factory";
const sessionService = createServiceProxy(SessionServiceProxy);
```

### Table Pattern (Ant Design + useDataTable)
```tsx
const fetchFn = useCallback(
  (skip, max, sort) => someService.getAll(new GetAllInput({ skipCount: skip, maxResultCount: max, sorting: sort })),
  [filters]
);
const { records, loading, pagination, handleTableChange, fetchData } = useDataTable<SomeListDto>(fetchFn);

<Table
  rowKey="id"
  dataSource={records}
  columns={columns}
  loading={loading}
  pagination={pagination}
  onChange={(pag, _, sorter) => handleTableChange(pag, sorter)}
/>
```

### Modal Pattern (Ant Design)
```tsx
<Modal
  title={entityId ? L("Edit") : L("CreateNew")}
  open={visible}
  onCancel={onClose}
  width={900}
  footer={[
    <Button key="cancel" onClick={onClose}>{L("Cancel")}</Button>,
    <Button key="save" type="primary" onClick={handleSave}>{L("Save")}</Button>,
  ]}
>
  {/* form content */}
</Modal>
```

Parent controls `visible` state and passes `onClose`/`onSaved` callbacks.

### Permission Pattern
```tsx
const { isGranted } = usePermissions();

// Conditional rendering
{isGranted("Pages.Administration.Users.Create") && (
  <Button type="primary" onClick={openCreateModal}>{L("CreateNewUser")}</Button>
)}

// Menu filtering (appNavigation.tsx) uses permissionName on menu items
```

### Localization Pattern
```tsx
import { L } from "@/lib/L";
// L() wraps window.L which is populated from ABP's /AbpUserConfiguration/GetAll
<span>{L("Users")}</span>
<span>{L("XItemsDeleted", deletedCount)}</span>
```

### Routing Pattern (lazy-loaded)
```tsx
// routes/AppRouter.tsx
const SomePage = React.lazy(() => import("../pages/admin/some-feature"));

<Route path="some-feature" element={
  <ProtectedRoute><SomePage /></ProtectedRoute>
} />
```

### State Management (Redux Toolkit)
Four slices in `app/slices/`:
- **authSlice** — `{ user, token, isAuthenticated, isLoading }`
- **sessionSlice** — `{ user, tenant, permissions, features, theme, isLoaded }`
- **uiSlice** — `{ theme, activeThemeName, containerClass, isLoading }`
- **localeSlice** — `{ selectedLang, messages, status }`

### Form Pattern
Simple forms use **react-hook-form**:
```tsx
const { register, handleSubmit, formState } = useForm<FormValues>({ mode: "onChange" });
```
Complex entity forms use **controlled state** with `useState` per field.

## Backend Connection

- Backend URL: `VITE_API_URL` env var → `AppConsts.remoteServiceBaseUrl`
- Auth: JWT Bearer token via Axios interceptor in `http-client.ts`
- Multi-tenancy: `Abp-TenantId` header from cookie
- Locale: `.AspNetCore.Culture` header
- ABP response envelope unwrapped automatically (`{ __abp: true, result }` → `result`)
- NSwag proxy regeneration: run `react/nswag/refresh.bat` (backend must be running)

## Constraints

- Functional components only (no class components)
- TypeScript required for all files (strict mode)
- Use Ant Design for UI components (Table, Modal, Form, Button, Tabs, etc.)
- Use Metronic for layout/theming structure
- Permission checks on all action buttons and pages
- Never manually edit `src/api/generated/service-proxies.ts`
- Use `L()` for all user-facing text
- Lazy-load all page routes with `React.lazy()`
